EXPLANATION:
- Column A: Main menu "Hospital"
- Column B: Bed - standalone table with inline list definitions
- Column C: BedRegistration - references Bed, C_BPartner
- Column D: PatientActivity - type of attention list
- Column E: Surgery - EXTENDS C_BPartner (Master=C_BPartner) - sub-detail to existing model!
- Column F: Treatment - child of Surgery (Master=Surgery) - 3-level hierarchy
- Column G: PatientLog - child of C_BPartner with calculations

================================================================================
SHEET: Callout (optional - for formula columns)
================================================================================
| Table      | SourceColumn | TargetColumn | Formula                          |
|------------|--------------|--------------|----------------------------------|
| PatientLog | QtyDosage    | Percentage   | QtyDosage / MaxDosage * 100      |
| PatientLog | Name         | Description  | C_BPartner.C_BP_Group.Name       |

CALLOUT SYNTAX (from original Ninja RO_ModelMaker.Callout field):
- SourceColumn > TargetColumn = Formula
- Formula types:
  1. Arithmetic: QtyDosage / MaxDosage * 100
  2. Traverse: C_BPartner.C_BP_Group.Name (follows FK relationships)
  3. Combined: ProductQty * UnitPrice (simple calculation)

================================================================================
EXPECTED TABLES CREATED
================================================================================
1. HO_Bed - Hospital beds with location info
   - Links to: None (standalone reference)

2. HO_BedRegistration - Patient bed assignments
   - Links to: C_BPartner (patient), AD_User, HO_Bed

3. HO_PatientActivity - Types of medical attention
   - Links to: None (standalone reference)

4. HO_Surgery - Surgery records
   - Links to: C_BPartner (patient) - EXTENDS EXISTING TABLE
   - This creates a sub-tab in Business Partner window!

5. HO_Treatment - Treatment records for surgery
   - Links to: HO_Surgery (master-detail)
   - 3-level hierarchy: C_BPartner -> HO_Surgery -> HO_Treatment

6. HO_PatientLog - Patient activity log with calculations
   - Links to: C_BPartner, HO_PatientActivity, HO_BedRegistration
   - Has Callout for auto-calculating Percentage

================================================================================
SPECIAL FEATURES DEMONSTRATED
================================================================================

1. EXTENDING EXISTING MODEL (Master=C_BPartner):
   Surgery table has C_BPartner as Master, meaning it will be added as a
   sub-tab under the Business Partner window. This is powerful for extending
   standard iDempiere entities without modifying core code.

2. MULTI-LEVEL MASTER-DETAIL:
   C_BPartner -> Surgery -> Treatment
   This creates a 3-level tab structure in the window.

3. INLINE LIST DEFINITIONS:
   Original Ninja supported: L#BedNo=1/2/3/4/5/6/7/8/9
   This creates the reference AND populates values in one definition.

4. CALLOUT FORMULAS:
   QtyDosage > Percentage = QtyDosage / MaxDosage * 100
   Auto-calculates fields based on other field changes.

5. REFERENCING EXISTING TABLES:
   C_BPartner_ID, AD_User_ID, M_Product_ID - uses existing iDempiere tables.
